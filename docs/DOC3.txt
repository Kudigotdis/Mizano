All views stack inside overlay divs. The back chevron always returns to the previous level.\nDependency\nDocuments 1 and 2 must be fully built and tested before beginning this document&apos;s sessions.\n\n\n\nPART 13: THE UNIVERSAL EDIT PATTERN\n\nThis pattern applies identically to ALL 7 entity types. No exceptions. No special cases. Every entity has one owner (or set of admins), and only they see the Edit chip.\n\n\n\n1\nUser views their entity in a detail card or sub-panel (e.g. their Group in MyGroups sub-panel)\n\n2\nThey see an &apos;Edit&apos; action chip — visible ONLY if userId matches the entity&apos;s owner_uid or is in the entity&apos;s admins[] array\n\n3\nUser taps Edit\n\n4\nA blue sticky bar slides in at the BOTTOM of the overlay: [Save Changes] button on left, [Cancel] on right. This is the EDITING bar.\n\n5\nAll text content in the card transforms in-place into editable inputs — text to &lt;input&gt;, paragraphs to &lt;textarea&gt;, chips to toggleable chips, photos to tap-to-replace zones\n\n6\nUser edits. Nothing writes to IndexedDB until Save is tapped.\n\n7\nUser taps Save Changes → client-side validation runs (required fields, formats, uniqueness where needed)\n\n8\nVALIDATION PASS: window.mizanoStorage.updateEntity(storeName, localId, updates) fires → last_modified = Date.now() → sync_status = &apos;pending&apos; → EDITING bar disappears → entity re-renders in-place with updated data → toast: &apos;Changes saved.&apos;\n\n9\nVALIDATION FAIL: Specific fields highlighted red, error text shown below each failed field. EDITING bar stays visible. Scroll to first error.\n\n10\nUser taps Cancel → all input fields revert to original values. Zero writes to IndexedDB. EDITING bar disappears. Entity shows original data.\n\n\n\nNON-NEGOTIABLE: The creator of any entity is ALWAYS able to edit it. There is no entity in Mizano that the creator cannot edit after creation.\n\n\n\n\n13.1 Edit Permission Matrix\nEntity\nWho Can Edit\nEdit Chip Visibility Rule\nUser Profile\nThe profile owner only\nuserId === profile.uid\nMinor Profile\nThe guardian who created it only\nuserId === minor.guardian_uid\nEvent\nThe organiser (user or group admin)\nuserId === activity.organizer_uid OR userId in organizer_group.admins\nGroup\nOwner and any Admin role members\nuserId === group.owner_uid OR userId in group.admins[]\nAssociation\nPrimary admin (creator) and added admins\nuserId === association.primary_admin OR userId in association.admins[]\nVenue\nSubmitter and linked org admin\nuserId === venue.submitted_by OR userId in linked_entity.admins[]\nBusiness\nOwner and co-admin users\nuserId === business.owner_uid OR userId in business.co_admins[]\nPlayer File\nThe file creator only\nuserId === player_file.creator_uid\n\n\n\nPART 14: MY GROUPS — Complete Sub-Panel Spec\n\nFile: MyGroups.js.
