## 🎯 DESIGN CONSISTENCY CHECKLIST

For each new prompt, verify these elements are specified:

### Visual Design
- [ ] Background color (default: White #FFFFFF)
- [ ] Border colors with 70% opacity
- [ ] Typography (Inter/Roboto, 14px base)
- [ ] Touch targets (minimum 44px)
- [ ] Spacing (8px grid system)
- [ ] No shadows or gradients (except where noted)
- [ ] Color-coded borders for different content types
- [ ] Grayscale wireframe aesthetic

### Navigation
- [ ] Back arrow position (top-left)
- [ ] Contextual actions (top-right)
- [ ] Bottom menu bar (if main page)
- [ ] Swipe gestures defined
- [ ] Tap actions specified
- [ ] "Previous page" return logic

### Offline-First
- [ ] Local storage strategy (IndexedDB)
- [ ] Sync queue behavior
- [ ] Offline indicator placement
- [ ] Manual sync option
- [ ] Auto-save frequency
- [ ] Conflict resolution approach

### Guardian Safety (if U16 content)
- [ ] Guardian approval logic
- [ ] Security logging
- [ ] Privacy banner
- [ ] Contact visibility rules

### Data Efficiency
- [ ] WhatsApp/Facebook deep links (not internal messaging)
- [ ] WebP image format specified
- [ ] Text-only options available
- [ ] Lazy loading strategy
- [ ] No heavy video embeds

### Component Consistency
- [ ] +/- toggle system for collapsible sections
- [ ] Card design (border, radius, padding)
- [ ] Button styles (primary, secondary, text)
- [ ] Form field styles
- [ ] Badge/status indicator colors
- [ ] Empty state designs

## ⚠️ IMPORTANT REMINDERS

1. **Offline-First Philosophy:** Every prompt MUST specify offline behavior, sync logic, and local storage strategy.

2. **Guardian Safety:** Any page accessible to U16 users MUST include Guardian approval logic and security logging.

3. **Data Efficiency:** All external links MUST use deep linking (WhatsApp, Facebook). NO internal heavy features.

4. **+/- Toggle System:** ALL collapsible sections use consistent +/- or ▶/▼ icons.

5. **Color-Coded Borders:** Status badges and activity states MUST use the defined color system (Orange: Active Soon, Green: Active Now, Gray: Passed, etc.).

6. **Touch Targets:** ALL interactive elements MUST meet 44px minimum.

7. **WebP Images:** ALL image references MUST specify WebP format for data savings.

8. **Bluetooth Sync:** Equipment Ledger and offline match features MUST specify Bluetooth sync option.
