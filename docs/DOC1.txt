Every single write goes to IndexedDB first, locally on the user&apos;s device. Google Sheets is a background sync target, not the primary data store. The app must function completely with no internet connection.1.1 The Three Storage LayersIndexedDBPRIMARY STORE. All user-created entities (groups, events, businesses, etc.) live here. Persists across page reloads. Queried directly by all JS modules. This is the single source of truth for the app.LocalStoragePreferences ONLY. Dark mode toggle, last visited panel, notification sound toggle. No entity data, no profile data, no session tokens are stored here.Google Sheets API v4Cloud sync master. Background push every 15 minutes, or triggered manually by user. Conflict resolution rule: if cloud.last_modified &gt; local.last_modified + 5 seconds, cloud wins. Pull updates on app launch when online.ABSOLUTE RULE: Medical data — injuries, allergies, chronic conditions, any health information — is stored in IndexedDB ONLY with a &apos;medical_records&apos; store. This data NEVER appears in any sync payload sent to Google Sheets. Not ever. Not for any reason.1.2 The 6 Mandatory Sync Metadata FieldsEvery IndexedDB object store that syncs to Google Sheets MUST include all 6 of these fields on every single record. If any field is missing, the sync queue will silently fail.6 MANDATORY SYNC FIELDS — All 6 Required on Every Synced Recordlocal_id — Auto-increment integer. Primary key on the device. Never changes after creation.cloud_id — String. Google Sheets row ID. Null until the first successful sync to the cloud.last_modified — Unix timestamp in milliseconds. Updated every time the record is changed locally.sync_status — String enum: &apos;pending&apos; | &apos;synced&apos; | &apos;conflict&apos; | &apos;error&apos;. Set to &apos;pending&apos; on every local write.sync_attempts — Integer. Counts failed push attempts. If it reaches 3, set sync_status to &apos;error&apos; and alert the user.created_at — Unix timestamp in milliseconds. Set once on creation. Never changes.1.3 Complete IndexedDB Object Store RegistryStorageManager.js must create ALL of the following stores in the db.onupgradeneeded handler.
